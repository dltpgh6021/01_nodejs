# 01 백엔드 개발이 뭐에요? 

## 01-1 백엔드 개발 기본 개념 다지기

### 이 책에서 공부할 백엔드 개발 기초 지식

1. 자바스크립트에 기반한 노드 
node는 언어가 아니라 자바스크립트라는 언어로 서버를 개발할 수 있도록 도와주는 도구. 
백엔드 개발에서 자바스크립트를 사용할 수 있게 환경을 만들어 주는 것. 

2. 익스프레스 프레임워크
프레임워크란? 서버를 만들 때 반복되는 패턴과 복잡한 기능을 처리해 주는 함수를 제공함. 

3. 데이터베이스 관리를 위한 몽고DB
자료를 저장하는 데이터베에스가 당연히 중요. 
이 책에서는 NoSQL데이터베이스인 몽고DB를 사용하여 다룰것임. 

4. API 구축하기
백엔드 개발의 핵심은 API를 구축하는 것. API = Application Programming Interface
API는 서로 다른 2가지 프로그램을 중간에서 연결해 주는 프로그램이라고 생각하면 쉽다. 
서로 다른 2개의 언어 사이에서 의사 소통을 원활하게 해주는 통역사나 번역 애플리케이션이 API이다. 
웹 애플리케이션을 만들 때는 클라이언트와 서버 사이에서 자료를 주고받아야 함.
이때, API가 클라이언트와 서버 사이에 상호 작용을 할 수 있도록 만들고 자료를 전달해주는 역할을 한다. 

## 01-2 노드란 무엇일까요? 

### 노드의 정의
Node.js는 Google Chrome V8 자바스크립트 엔진을 실행하는 JavaScript 런타임입니다. 

크롬 브라우저의 V8엔진
 - 개발자가 작성한 코드를 이해할 수 있는 프로그램이 웹 브라우저 안에 포함되어 있는 것. 

자바스크립트 런타임
 - 런타임이란? 자바스크립트를 실행할 수 있는 환경 정도로 이해하기. 

### 노드의 장점 3가지

1. 하나의 언어로 프런트엔드와 백엔드를 개발할 수 있습니다. 
처음에 웹 개발시 프런트엔드와 백엔드의 역할이 구분되어있었지만, 노드의 등장으로 인해 자바스크립트를 사용할 수 있게 되었다. 

2. 개발자 커뮤니티의 규모가 크고 다양하다. 
노드는 2009년에 출시했는데 지금까지 사용되고 있다. 
그동안 개발자들 커뮤가 많아져서 규모가 큼. 

3. 서브파티 모듈이 많다. 
노드는 사용자가 많고, 오랫동안 업데이트해 온 만큼 사용할 수 있는 서드파티 모듈이 많다. 
서드 파티 모듈이란 노드 제작 업체와 개발자 외 다른 사람이 만들어 놓은 기능 묶음이라고 생각하면 된다. 
개발을 하다보면 필요한 기능이 많은데, 이미 다른 사람들이 만들어 놓은 모듈이 많으니 끌어다 쓰면 됨.

## 01-4 패키지 관리는 npm으로 해요

### 모듈, 패키지, 라이브러리

모듈 : 프로그램을 기능별로 쪼개 필요할 때 마다 가져와서 사용하는 단위.
패키지 : 자주 사용하는 기능 모듈들을 묶어 놓은 것. 
라이브러리 : 모듈보다 큰 단위. 특정 기능을 수행할 수 있도록 모듈을 여러 개 묶어서 하나의 라이브러리로 만들어 둔다. 

### 패키지 매니저 - npm
패키지 매니저는 애플리케이션을 개발할 때 다양한 패키지를 관리할 수 있게 해주는 편리한 도구이다.

# 07 익스프레스 도우미, 미들웨어

## 07-1 자주 사용하는 미들웨어

### 미들웨어란? 
: 요청과 응답 중간에 있으면서 요청을 처리하거나 원하는 형태로 응답을 수정하는 함수. 

Ex) 로그인 처리 과정
1. 요청 안에 포함된 아이디와 비밀번호의 값을 애플리케이션에서 읽을 수 있는 형태로 변환함. (Parsing 파싱)
2. 아이디와 비밀번호의 값을 사용해서 사용자 인증
3. 처리 결과를 다음 단계로 넘겨준다. 예를 들어, 렌더링하여 화면에 보여줄 수 있다. 

**미들웨어의 역할** 
일반적으로 다음과 같은 일을 처리한다. 
- 요청 전처리 : 요청이 서버에 도착하기 전에 실행하는 작업. 사용자 인증이나 폼 내용 검증 등의 작업 처리. 
- 라우팅 처리 : 라우트 코드를 좀 더 읽기 쉽고 관리하기 쉽도록 모듈화 하는 것. 
- 응답 처리 : 서버에서 클라이언트로 응답을 보낼 때 자료를 적절한 형태로 변환하거나 오류를 처리하는 작업을 한다. 

**애플리케이션 미들웨어**
애플리케이션 단계에서 사용하는 미들웨어
애플리케이션 자체에 get 함수나 post 함수를 사용해서 미들웨어를 처리하면 애플리케이션 레벨 미들웨어라고 함. 

**라우터 미들웨어** 
애플리케이션 규모가 커지면 처리해야 할 라우트가 많아지고 길고 복잡해짐. 
라우터 객체를 통해 라우트 코드를 정리할 수 있음. 

익스프레스의 라우터 객체를 사용하기 위해 router라는 인스턴스 객체를 만들어야 함. 
이렇게 하면 묶어서 표현 가능 

```js
const router = express.Router();

router.route("/contacts")
    .get((req, res) => { ... })
    .post((req, res) => { ... })
```

**app.use 함수**
애플리케이션에서 미들웨어를 사용하려면 어떤 것을 사용할 것인지 알려줘야 함. 
이것을 '미들웨어를 등록한다' 라고 표현함. app.use 함수는 애플리케이션에서 실행할 미들웨어를 등록할 때 사용함. 

app.use에서는 '경로' 옵션을 사용할 수 있음. 경로를 따로 지정하지 않으면 애플리케이션 전체에서 해당 미들웨어를 사용하겠다는 뜻이고, 경로를 지정하면 그 경로로 요청할 때만 해당 미들웨어를 사용한다는 뜻.

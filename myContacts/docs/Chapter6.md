# 06 익스프레스로 더 쉽게 서버 만들기
## 06-1 익스프레스 시작하기

### 왜 익스프레스인가
- 라우팅 : if/switch문보다 간결한 라우팅 방식
- 미들웨어 : 요청과 응답 사이 여러 기능 실행 가능
- 템플릿 엔진 : HTML 페이지는 기본적으로 정적이지만, 서버와 함께 사용해서 동적인 HTML 페이지 만들 수 있음
- 정적인 파일 지원 : CSS나 JS 파일, 이미지 처럼 정적인 파일을 수비게 서비스 할 수 있는 기능도 제공함. 

**한걸음 더**
package.json 파일에 "scripts" 부분이 있다. 
이 안에 "start": "nodemon app.js"를 적어주면, npm start만 해도 nodemon app.js 명령어가 자동 실행된다. 

"start" 라는 명령을 하면 다음과 같은 스크립트를 실행해라 라는 뜻이다. 
나중에 배포를 하게 된다면, nodemon이 아닌, node로 바꿔서 배포해야 한다. 

## 06-2 익스프레스에서 라우팅하기

### HTTP 모듈 vs 익스프레스 라우팅
if, switch문으로 인해 코드 길어짐 -> 가독성 낮아짐

app.METHOD(path, handler) 함수로 편리해질 수 있음
- app : 익스프레스로 만든 인스턴스
- METHOD : 요청에서 사용하는 메서드
- paht : 요청 URL
- handler : 실행할 콜백 함수

### 라우트 파라미터
실행 파일 내에서 /contacts 경로는 모든 연락처 정보를 가져와서 보여줄 때 사용함. 
아이디값을 사용해 필요한 연락처만 가져오려면 어떻게 해야 하는가? 

**/요청 URL/:id** 와 같은 형식을 사용하면 된다. 
/contacts/1
/contacts/10
이와 같이 사용하면 라우트 파라미터로 동적 URL 처리가 가능해진다. 


## 06-3 익스프레스의 요청 객체와 응답 객체

### 요청 객체의 주요 속성 살펴보기
req.body : 서버로 post 요청할 때 넘겨준 정보를 담고 있다. 
req.cookies : 클라이언트에 저장된 쿠키 정보를 서버로 넘겼을 경우 쿠리 정보를 담고 있음.
req.headers : 서버로 요청 보낼 때 같이 보낸 헤더 정보를 담고 있음. 
req.params : 파라미터를 갖고있을 경우 파라미터 정보 담고 있음. 
req.query : 검색어와 관련된 질의 매개변수가 저장됨. 

### 응답 객체에서 사용하는 함수
res.download : 파일을 내려받는다
res.end : 응답 프로세스를 종료
res.json : JSON 응답을 전송합니다. 
res.jsonp : JSONP 지원을 통해 JSON 응답을 전송함
res.redirect : 요청 경로를 재지정해서 강제이동
res.render : 뷰 템플릿을 화면에 렌더링
res.send : 어떤 유형이든 res.send()내의 내용 전달
res.sendFile : 지정한 경로의 파일을 읽어 내용 전송
res.sendStatus : 상태 메시지와 함께 HTTP 상태 코드를 전송함
res.status : 응답 상태의 코드를 설정. 

**클라이언트로 보내기 - res.send 함수**
문자열이나 객체, 배열 등 괄호 안의 내용을 클라이언트로 전송하는 함수
Content-Type 헤더를 자동으로 만들고 내용을 JSON 형식으로 변환하거나 일반 텍스트로 전송

**JSON 형식으로 변환하기 - res.json 함수**
괄호 안의 객체나 배열을 JSON 형식으로 변환해서 전송한다. 

**파일을 클라이언트로 보내기 - res.sendFile 함수**
앞에서는 간단히 텍스트 문자열을 화면에 표시해보면서 라우팅을 어떻게 처리하는지 알아보았다. 
그러나 애플리케이션을 만들 때에는 라우팅에서 특정 HTML 문자열을 가져와 보여줘야 하는 경우도 많다. 
이 함수를 통해 웹 페이지에 HTML 화면 띄울 수 있음. 

**상태 코드 설정하기 - res.status**
애플리케이션을 만들 때 반드시 설정해야 하는 것은 아니다. 
애플리케이션마다 상태코드를 확인해서 그에 따라 다르게 처리해야 하는 경우가 있는데, res.status 함수를 사용해서 상태 코드를 설정한다.
